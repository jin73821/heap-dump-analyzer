1. 개요
------------------------------------------------------------------------
Settings 페이지(/settings)에 LLM API 연동 설정 카드를 추가하고,
분석 결과 페이지(analyze.html)에 AI 분석 요청 선택지를 표시한다.

핵심 변경:
  - 자동 LLM 호출 없음 → 사용자가 명시적으로 "AI 분석" 버튼 클릭
  - 분석 결과 페이지 헤더에 AI 분석 여부 배지 표시
  - AI 미분석 상태는 회색(비활성) 표시
  - 다중 LLM 프로바이더 지원 (Claude, GPT, Genspark 등)


2. 다중 프로바이더 아키텍처
------------------------------------------------------------------------

2-1. 프로바이더별 차이점
........................................................................
  ┌──────────┬──────────────────────────────┬───────────────────┬──────────────┐
  │ Provider │ API Endpoint                 │ 인증 헤더          │ 요청 형식     │
  ├──────────┼──────────────────────────────┼───────────────────┼──────────────┤
  │ Claude   │ api.anthropic.com/v1/messages│ x-api-key         │ Messages API │
  │ GPT      │ api.openai.com/v1/chat/...   │ Authorization:    │ Chat API     │
  │          │                              │ Bearer <key>      │              │
  │ Genspark │ (사용자 지정 URL)             │ Authorization:    │ Chat API     │
  │          │                              │ Bearer <key>      │ (호환)       │
  │ Custom   │ (사용자 지정 URL)             │ (사용자 지정)      │ (자동 감지)  │
  └──────────┴──────────────────────────────┴───────────────────┴──────────────┘

2-2. 추상화 전략 — LlmProvider 인터페이스
........................................................................
  인터페이스를 사용하지 않고, LlmAnalysisService 내부에서 provider 값에
  따라 요청 형식을 분기하는 방식 채택.

  이유:
  - 현재 프로젝트가 인터페이스/DI 패턴을 사용하지 않는 단순 구조
  - 프로바이더 간 차이는 (1) HTTP 헤더 (2) JSON 요청/응답 포맷 뿐
  - 별도 클래스 분리 시 과도한 복잡성 → 메서드 분기로 충분

  구현:
  LlmAnalysisService 내:

    private HttpURLConnection buildRequest(String prompt) {
        switch (llmProvider) {
            case "claude":  return buildClaudeRequest(prompt);
            case "gpt":     return buildGptRequest(prompt);
            case "genspark": return buildGensparkRequest(prompt);
            default:        return buildOpenAICompatibleRequest(prompt);
        }
    }

    private Map<String,Object> parseResponse(String provider, String body) {
        switch (provider) {
            case "claude":  return parseClaudeResponse(body);
            case "gpt":     return parseGptResponse(body);
            case "genspark": return parseGptResponse(body);  // OpenAI 호환
            default:        return parseGptResponse(body);    // 기본: OpenAI 호환
        }
    }

2-3. 프로바이더별 요청 형식
........................................................................
  [Claude — Messages API]
    POST https://api.anthropic.com/v1/messages
    Headers:
      x-api-key: {apiKey}
      anthropic-version: 2023-06-01
      content-type: application/json
    Body:
      {
        "model": "claude-sonnet-4-20250514",
        "max_tokens": 4000,
        "system": "...",
        "messages": [{"role":"user", "content":"..."}]
      }
    Response:
      { "content": [{"type":"text","text":"..."}],
        "model": "...", "usage": {"input_tokens":N, "output_tokens":N} }

  [GPT — Chat Completions API]
    POST https://api.openai.com/v1/chat/completions
    Headers:
      Authorization: Bearer {apiKey}
      content-type: application/json
    Body:
      {
        "model": "gpt-4o",
        "max_tokens": 4000,
        "messages": [
          {"role":"system", "content":"..."},
          {"role":"user", "content":"..."}
        ]
      }
    Response:
      { "choices": [{"message":{"content":"..."}}],
        "model": "...", "usage": {"prompt_tokens":N, "completion_tokens":N} }

  [Genspark / Custom — OpenAI 호환]
    GPT와 동일한 Chat Completions 형식, URL만 다름
    POST {사용자 지정 URL}
    Headers: Authorization: Bearer {apiKey}
    Body/Response: GPT와 동일

  │ claude   │ claude-sonnet-4-20250514 (추천)                │
  │          │ claude-haiku-4-5-20251001 (빠름/저비용)         │
  │          │ claude-opus-4-20250514 (최고 성능)              │
  ├──────────┼────────────────────────────────────────────────┤
  │ gpt      │ gpt-4o (추천)                                  │
  │          │ gpt-4o-mini (빠름/저비용)                       │
  │          │ gpt-4-turbo                                    │
  ├──────────┼────────────────────────────────────────────────┤
  │ genspark │ (사용자 직접 입력)                              │
  ├──────────┼────────────────────────────────────────────────┤
  │ custom   │ (사용자 직접 입력)                              │
  └──────────┴────────────────────────────────────────────────┘

  Settings에서 Provider 변경 시:
    - 모델 드롭다운 옵션이 해당 프로바이더 목록으로 변경
    - API URL이 기본값으로 자동 채움 (사용자 수정 가능)
    - genspark/custom은 모델명 직접 입력 가능 (텍스트 input)

5-2. Provider 변경 시 동작 (JavaScript)
........................................................................
  function onProviderChange() {
      var provider = document.getElementById('llmProviderSelect').value;

      // (1) API URL 기본값 채움
      var urlInput = document.getElementById('llmApiUrlInput');
      var defaultUrls = {
          'claude':  'https://api.anthropic.com/v1/messages',
          'gpt':     'https://api.openai.com/v1/chat/completions',
          'genspark': '',
          'custom':  ''
      };
      urlInput.value = defaultUrls[provider] || '';
      urlInput.readOnly = false;  // 모든 프로바이더에서 편집 가능

      // (2) 모델 드롭다운 ↔ 텍스트 입력 전환
      var modelSelect = document.getElementById('llmModelSelect');
      var modelInput = document.getElementById('llmModelInput');
      var providerModels = {
          'claude': [
              {value:'claude-sonnet-4-20250514', label:'Claude Sonnet 4 (추천)'},
              {value:'claude-haiku-4-5-20251001', label:'Claude Haiku 4.5 (빠름)'},
              {value:'claude-opus-4-20250514', label:'Claude Opus 4 (최고 성능)'}
          ],
          'gpt': [
              {value:'gpt-4o', label:'GPT-4o (추천)'},
              {value:'gpt-4o-mini', label:'GPT-4o Mini (빠름)'},
              {value:'gpt-4-turbo', label:'GPT-4 Turbo'}
          ]
      };

      if (providerModels[provider]) {
          // 드롭다운 표시
          modelSelect.style.display = '';
          modelInput.style.display = 'none';
          modelSelect.innerHTML = '';
          providerModels[provider].forEach(function(m) {
              var opt = document.createElement('option');
              opt.value = m.value; opt.textContent = m.label;
              modelSelect.appendChild(opt);
          });
      } else {
          // 텍스트 입력 표시
          modelSelect.style.display = 'none';
          modelInput.style.display = '';
          modelInput.value = '';
          modelInput.placeholder = '모델 ID를 입력하세요';
      }
  }

5-3. 설정 저장 시 provider/apiUrl 포함
........................................................................
  function saveLlmConfig() {
      var provider = document.getElementById('llmProviderSelect').value;
      var modelSelect = document.getElementById('llmModelSelect');
      var modelInput = document.getElementById('llmModelInput');
      var model = modelSelect.style.display !== 'none'
                  ? modelSelect.value : modelInput.value.trim();
      var apiUrl = document.getElementById('llmApiUrlInput').value.trim();

      if (!apiUrl) { toast('API URL을 입력하세요', 'error'); return; }
      if (!model) { toast('모델을 선택하세요', 'error'); return; }

      fetch('/api/llm/config', {
          method: 'POST',
          headers: {'Content-Type':'application/json'},
          body: JSON.stringify({
              provider: provider,
              apiUrl: apiUrl,
              model: model
          })
      })...
  }

8. 검증 방법
------------------------------------------------------------------------
  (1) 빌드: mvn clean package -DskipTests && bash restart.sh

  (2) Settings — Provider 전환:
      - Claude 선택 → URL 자동 채움, 모델 드롭다운 3개
      - GPT 선택 → URL 자동 채움, 모델 드롭다운 3개
      - Genspark 선택 → URL 비움, 모델 텍스트 입력
      - Custom 선택 → URL 비움, 모델 텍스트 입력

  (3) Settings — 연결 테스트:
      - 각 프로바이더별 API 키 + 연결 테스트 성공/실패 확인

  (4) 분석 결과 페이지 — AI 미분석:
      - 헤더 회색 배지, AI 패널 회색 + 버튼 표시

  (5) 분석 결과 페이지 — AI 분석 실행:
      - 버튼 클릭 → 스피너 → 완료 시 파란 배지 전환

  (6) 영속화:
      - settings.json 내 provider, apiUrl, model 저장 확인
      - 서버 재시작 후 설정 유지 확인
